# pdf-service

A Python/FastAPI microservice that manages PDF document access based on the user's subscription plan.

## Description

`pdf-service` enforces reading limits based on the subscription plan passed via `X-User-Plan` header from the API gateway. It stores PDFs in S3 or Azure Blob Storage and returns time-limited pre-signed URLs for download.

### Access control rules

| Plan | PDFs per day | Reading time per day |
|---|---|---|
| Free | 1 | 30 minutes |
| Basic | 1 | Unlimited |
| Premium | Unlimited | Unlimited |

### Key design decisions
- **Stateless access control** — limits enforced via Redis counters with 24h TTL (auto-reset daily)
- **Pre-signed URLs** — clients never get direct storage credentials, only time-limited download links
- **Session-based tracking** — reading time counted from open to close, persisted to MySQL on close
- **Storage abstraction** — switch between S3 and Azure Blob via a single env var

## API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/pdfs` | List all available PDFs |
| `GET` | `/pdfs/{id}` | Get PDF metadata |
| `POST` | `/pdfs/{id}/open` | Open reading session (returns pre-signed URL) |
| `POST` | `/pdfs/{id}/close` | Close session (persists duration to MySQL) |
| `GET` | `/health` | Kubernetes probe |

## Getting Started

### Consumers
The following clients call this service via the api-gateway proxy:
- **frontend** (Vue 3 SPA) — `GET /pdfs`, `POST /pdfs/{id}/open`, `POST /pdfs/{id}/close`
- **api-gateway** — forwards requests with `X-User-ID`, `X-User-Plan` headers used for access control

### Requirements
- Python 3.12+
- MySQL
- Redis
- S3 bucket or Azure Blob container with PDFs uploaded

### Installation

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# Fill in your credentials
```

### Run locally

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

API docs available at `http://localhost:8000/docs` (auto-generated by FastAPI).

### Tests

```bash
pip install pytest
pytest tests/ -v
```

## Local Kubernetes Deployment

```bash
# Build
docker build -t pdf-service:latest .
docker save pdf-service:latest | docker exec -i $(docker ps -qf "name=desktop-control-plane") ctr -n k8s.io images import -

# Deploy
kubectl apply -f k8s/local/

# Verify
kubectl get pods -l app=pdf-service
kubectl logs -l app=pdf-service
```

## Environment Variables

| `DB_HOST` | MySQL host | `pdf-service-mysql` |
